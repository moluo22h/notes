# 敏捷开发概念模型

## 工作时间表

| 时间 | 工作                                | 描述 |
| ---- | ----------------------------------- | ---- |
| 周五 | 需求分析澄清、任务分配              |      |
| 周一 | 任务分配、开发设计                  |      |
| 周二 | 开发设计                            |      |
| 周三 | 设计评审                            |      |
| 周四 | 功能开发                            |      |
| 周五 | 功能开发                            |      |
| 周六 | 功能开发                            |      |
| 周一 | 代码评审及功能测试、单元测试        |      |
| 周二 | API文档、帮助文档、单元测试         |      |
| 周三 | API评审、帮助文档评审、单元测试评审 |      |
| 周四 | 版本发布                            |      |

## 工作详情

需求分析澄清

任务分配（工作录入、工作任务、开发者、审核者）

开发设计（数据库设计、接口设计、设计文档）

设计评审（数据库表名、字段名、字段类型、索引；接口风格）

功能开发

代码评审及功能测试（评审内容：类名、方法名、代码注释、复用模块、访问修饰符）

API文档

API评审（评审内容：API描述、参数说明、请求示例、返回示例、错误码）

帮助文档

帮助文档评审（流程、错别字）

单元测试

单元测试评审（模块单元测试覆盖率、功能单元测试、异常单元测试）

版本发布（发布申请、公有云数据备份、本地测试、版本更新内容）升级脚本  上线测试

没有预留缓冲区和研究时间
